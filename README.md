# 국내 주식 스윙(눌림목) 추천 시스템

상승 추세인 종목이 **20일 이동평균선까지 눌렸다가 반등하는 첫 자리**(눌림목)를
자동으로 스캔하고, 향후 **텔레그램 알림 + 웹 대시보드**로 실시간 추천하기 위한
프로젝트입니다.

> ⚠️ 투자 참고용 신호일 뿐 매매 권유가 아닙니다. 실제 투자 전 반드시 백테스트로
> 검증하고 본인 책임하에 판단하세요.

# 🌙 종가매매법(오버나이트) 추천 — 새로 추가됨

> **"장 마감 직전(15:20)에 사서, 다음날 아침 갭상승에 판다."** 갭상승한 종목이
> 전날 어떤 특징을 보였는지 과거 데이터로 검증하고, 그 특징을 만족하는 오늘의 후보를
> 매일 스캔해 **카카오톡으로 공유 가능한 웹페이지**로 공개합니다.

| 구성 | 파일 |
|------|------|
| 종가매매 전략·신호 (강한마감·거래량·돌파·과열필터) | `overnight/strategy.py` |
| 오버나이트 백테스트 (종가매수 → 다음시가매도) | `overnight/engine.py` |
| **갭상승 특징 분석** (전날 무엇이 달랐나 · 규칙 유효성) | `overnight/study.py` |

| 무결성 테스트 | `tests/test_overnight.py` |

## 이 매매법이 뭔가요?

국내 주식 정규장은 **09:00 시작 ~ 15:30 마감**입니다(15:20~15:30은 동시호가).
종가매매법은 **15:20쯤 강하게 마감하는 종목을 매수**해서 하룻밤 들고, **다음날
시초가(09:00)의 갭상승에 매도**하는 초단기(1박) 전략입니다.

### 갭상승 종목의 '전날' 특징 — 규칙으로 옮긴 것
과거 데이터에서 **다음날 갭상승한 종목**은 전날 아래 특징이 평균적으로 더 강했습니다.
이를 매수 규칙으로 만들었습니다(모두 종가 확정 후 판단 → 미래참조 없음):

1. **강한 양봉** — 당일 상승률 ≥ +2%, 종가 > 시가
2. **고가권 마감** — 종가가 당일 (고−저) 범위의 상단 60%+ 에서 마감
3. **거래량 급증** — 당일 거래량 ≥ 20일 평균 × 1.5
4. **돌파** — 종가가 최근 20일 고가를 상향 돌파
5. **단기 정배열** — 5일선 > 20일선, 종가 > 5일선
6. **과열 아님** — RSI < 80

규칙·수치는 `overnight/strategy.py` 의 `ClosingParams` 에서 조정합니다.
비용은 왕복 0.25%(매수·매도 수수료 + 거래세 + 슬리피지)로 반영했습니다.

## 실행 방법

```bash

# 2) 오프라인 검증 (합성 데이터, 네트워크 불필요)

# 테스트
```

출력물:

- `results/overnight_study.json` — 분석·성과 지표 (대시보드가 읽음)
- `state/overnight_latest.json` — **오늘의 종가매매 후보** (대시보드/알림용)
- `overnight_trades.csv` — 개별 오버나이트 매매 내역

### 📊 승률·기대수익률은 어디서 나오나
- **합성 데이터**: 엔진·분석 로직을 검증하기 위한 것이라 여기서 나오는 승률·기대수익률은
  **실제 시장 성과가 아닙니다.** (합성 데이터에 완만한 엣지를 일부러 주입해 놓았습니다.)
- **실데이터**(`--source real`): 회원님 PC나 GitHub Actions처럼 **네이버·KRX·야후 접근이
  열린 환경**에서 돌려야 진짜 승률·기대수익률이 나옵니다. 이 원격(웹) 세션은 보안 정책상
  증시 데이터 호스트가 막혀 있어 실데이터를 여기서 받을 수 없습니다.

> 💡 **현실적인 기대치**: 오버나이트 갭은 상당 부분 예측 불가능합니다. 실제로 이 유형의
> 엣지는 **건당 기대수익률이 얇고**(수십 bp 수준), 비용·슬리피지·거래불가(상한가 등)를
> 빼면 더 낮아집니다. 표본을 충분히 확보하고(수백~수천 건) 검증 구간에서도 (+)인지
> 반드시 확인하세요.

### ✅ 실데이터 검증 결과 (GitHub Actions 에서 실행)

**KOSPI 시총 상위 199종목 · 2019~2026(약 7.4년) · 매수 슬리피지 0.1% 반영.**
`tune_overnight.py` 로 데이터를 **학습(앞)/검증(뒤)** 으로 나눠 과최적화를 걸러낸 결과:

| 구분 | 표본 | 승률 | 건당 기대수익 | 손익비 | 연간 거래 |
|------|------|------|--------------|--------|-----------|
| **검증(미래 데이터)** | 864건 | **43.2%** | **+0.36%** | 1.86 | ≈355회 |
| 전체구간 참고 | 2,242건 | 44.2% | +0.22% | 1.69 | ≈298회 |

선택된 강건 조합: `up_min=0.03, close_pos_min=0.5, vol_mult=2.0, breakout_lookback=60, rsi_high=85`
(현재 `ClosingParams` 기본값). 갭상승 적중률 **30.2%** (무작위 16.3% 대비 **1.85배**).

**정직하게 읽어야 할 점**
- **승률은 43~45%로 절반 미만입니다.** 이기는 건보다 지는 건이 더 많습니다. 그런데도
  기대값이 (+)인 이유는 **이길 때(+1.4%)가 질 때(−1.0%)보다 크기 때문**(손익비>1)입니다.
  즉 **작은 손실을 자주 보고 가끔 크게 먹는** 구조라, 멘탈·손절 규율이 없으면 못 버팁니다.
- **건당 +0.36%는 매우 얇습니다.** 실제 체결 슬리피지가 조금만 더 커지거나 상한가로 못
  사는 종목을 빼면 엣지가 쉽게 사라질 수 있습니다.
- 단순 동일가중 포트폴리오의 **MDD는 −50%** 수준, 최악 단일매매 −21%. 갭 리스크는 실재합니다.
- **의외의 발견**: 실데이터에서 "고가권 마감"은 갭상승과 사실상 무관했습니다(갭상승 0.46 vs
  비갭상승 0.48). 실제로 다음날 갭을 예고한 건 **① 당일 큰 상승(+) ② 거래량 급증
  ③ 60일 신고가 돌파** 였습니다. (합성 데이터의 가정과 달랐고, 그래서 실데이터 검증이 중요합니다.)

> 이 표는 GitHub Actions 에서 `analyze.yml`(이 브랜치용) 을 실행해 나온 값입니다.
> `results/overnight_validation.json` 에 원본이 저장됩니다. 파라미터·기간을 바꿔 다시
> 돌리면 값도 갱신됩니다.

### ⏱️ "3시 10분까지 데이터로 3시 20분에 매수 — 10분 안에 분석 가능한가?"

**계산량은 전혀 문제가 안 됩니다.** 20일선·60일 신고가·20일 평균거래량 같은 **기준선은
장중에 안 변하므로 장 시작 전에 미리 계산**해 둡니다. 3시 20분엔 종목마다 *현재가·당일
거래량·당일 고저* 스냅샷을 기준선과 **비교만** 하면 되고, 이건 전 종목(2,000여 개)을
합쳐도 1초 안에 끝납니다. 진짜 필요한 건 연산이 아니라 **그 시각의 실시간 시세**입니다:

- 실시간 시세 소스: **한국투자증권(KIS) REST/WebSocket** 같은 증권사 오픈 API 로 15:10~15:20
  현재가·누적거래량을 받아 규칙과 대조 → 3:20 매수 후보 확정. (다음 단계로 붙일 수 있습니다.)
- ⚠️ 다만 이 백테스트는 **일봉**을 씁니다. 일봉의 종가는 15:30 확정가라, 15:20 매수를
  "종가로 샀다"고 본 것은 **미세한 낙관**입니다(그래서 슬리피지 0.1%를 더해 보수화했습니다).
  분(minute) 데이터로 15:20 스냅샷을 그대로 재현하면 더 정확하지만, 분봉은 확보가 더 어렵습니다.

### 눌림목 매수 규칙 (요약)

**진입 조건** (종가 확정 후 판단 → 다음 봉 시가 체결, 미래참조 없음)
1. 추세: 종가 > 60일선, 20일선 > 60일선(정배열), 60일선 우상향
2. 눌림: 당일 저가가 20일선 +2% 이내로 근접(단, 60일선은 안 깨짐), 직전 20일 고점이 현재가보다 8%+ 높음
3. 반등: 종가가 20일선 위로 회복 + 양봉 + RSI 40~68 + 거래량 ≥ 20일 평균×0.8

**청산 우선순위**: 손절 −5% → 익절 +10% → 20일선 −3% 이탈 → 15거래일 초과
왕복 비용(수수료+세금+슬리피지) 약 0.35% 반영.

모든 수치는 `swing/strategy.py`의 `PullbackParams`에서 조정할 수 있습니다.

### 텔레그램 봇 설정 (5분)

1. 텔레그램에서 **@BotFather** → `/newbot` → **봇 토큰**(`123456:ABC...`) 발급
2. 만든 봇과 대화 시작 후 **@userinfobot** 으로 본인 **chat_id**(숫자) 확인
3. 로컬 테스트:
   ```bash
   export TELEGRAM_TOKEN="봇토큰"
   export TELEGRAM_CHAT_ID="내chat_id"
   python run_scan.py --market KOSPI --top 100          # 실데이터 + 전송
   python run_scan.py --source synthetic --dry-run       # 미전송 테스트
   ```

### GitHub Actions 자동 실행 (무료)

1. 저장소 **Settings → Secrets and variables → Actions** 에 두 개 등록:
   `TELEGRAM_TOKEN`, `TELEGRAM_CHAT_ID`
2. 워크플로우가 **평일 장중(KST 09:00~15:30) 30분마다** 자동 스캔 → 신규 신호를 텔레그램 전송
3. 스캔 결과는 `state/` 에 커밋되어 중복 알림을 방지하고, 이후 웹 대시보드가 이를 읽습니다.

> ⚠️ **예약(schedule) 실행은 기본 브랜치(`main`)에 워크플로우가 있어야만 동작**합니다.
> 지금은 작업 브랜치에 있으므로, `main` 에 병합하면 자동 실행이 시작됩니다.

## 4단계: 전략 튜닝 (과최적화 방지) ✅

`tune.py` — 파라미터 조합을 격자 탐색하되, 데이터를 **앞(학습)/뒤(검증)** 로 나눠
**검증 구간에서도 성적이 유지되는 강건한 조합**만 상위로 올립니다. 학습에서만 좋고
검증에서 무너지는 조합은 과최적화이므로 걸러집니다.

```bash
# 실데이터 (네트워크 열린 환경) — 기간을 길게 줄수록 신뢰도 ↑
python tune.py --source real --market KOSPI --top 100 --start 2019-01-01

# 오프라인 검증
python tune.py --source synthetic --n 60 --days 1000
```

- 탐색 격자는 `tune.py`의 `GRID` 에서 조정 (조합 수가 곱으로 늘어나니 과하게 넓히지 말 것)
- 표본이 `MIN_TRADES`(기본 20) 미만인 조합은 신뢰 불가로 제외
- **고르는 법**: 검증(test) 기대값이 (+)이고 학습·검증 성적이 크게 어긋나지 않는 조합

### 튜닝 레버 (전략에 추가된 선택 옵션)

`PullbackParams` 에 ATR 기반 청산과 트레일링 스톱을 넣어 탐색 폭을 넓힐 수 있습니다
(기본은 비활성 = 기존 고정 % 청산):

| 파라미터 | 의미 |
|----------|------|
| `use_atr_exits` | 손절/익절을 진입시점 ATR 배수로 산정 |
| `atr_stop_mult` / `atr_target_mult` | ATR 손절/익절 배수 |
| `trail_atr_mult` | >0 이면 이익 구간에서 최고종가 대비 ATR 배수 이탈 시 청산 |
